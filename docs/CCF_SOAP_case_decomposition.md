# visit_timeline.csv 10건 SOAP 분해 — 반응형 차팅을 위한 의사 질문 시퀀스 추출

> 입력: `visit_timeline.csv` 의 첫 10건 (의미 있는 증상 텍스트가 있는 행만 선별)
> 스키마: `docs/CCF_SOAP_charting_requirements.md`
> 목적: **의사가 어떤 순서로 묻고 기록하는지**를 top-down으로 분해하여, 클릭 차팅에서 다음 단계를 추천하기 위한 시퀀스 패턴을 추출

## 표기 규칙

- **순서**: 차트 본문에 적힌 순서를 의사의 질문/사고 순서의 근사값으로 사용 (10건 표본 기준)
- **항목**: CCF 스키마의 항목명과 매핑 (없는 부위는 일반 항목으로 표기)
- **다음 질문 후보**: 그 항목을 채운 직후, 이 의사라면 무엇을 다음에 클릭/입력할지 추정
- **결측(—)**: 차트에 적히지 않은 항목 (의사가 묻지 않았거나, 음성이라 생략했을 가능성)

## 표본의 한계 (먼저)

- 10건은 **포맷 검증용 파일럿**입니다. 의사 식별 컬럼이 없어 동일 의사 가정으로 처리했습니다.
- 차팅 텍스트의 순서가 실제 질문 순서와 100% 일치하진 않습니다 (사후 보정 가능성). 다만 이 의사의 차팅은 흐름이 일관되어 근사로 사용 가능.
- 본격 트리거 그래프는 **12,284건 전체**를 부위별로 묶어 빈도 분석할 때 통계적 의미가 생깁니다.

| 7 | P | F/U | open FU (필요시 재방문) | — |

**관찰 포인트**: **"운동 직후 양측"** = 골절보다는 사용 과다(타박상) 시사 → x-ray + 초음파로 골절 배제하고 짧은 약처방 + open FU.

## 패턴 요약 (10건 표본 기준 — 통계 아닌 관찰)

### 1. S 단계의 표준 시퀀스

```
[내원 모드 분기]
├─ 호소 (Case 1,2,3,4,6,7,10)
├─ 추적 (Case 8) — "치료 경과 시점"부터
├─ 처치 (Case 5) — "내원 목적: 주사"부터
└─ 타원 수술 후 (Case 9) — "PO N주차"부터

[호소 모드 표준 흐름]
1. 호소 부위 + 측 (양측/좌/우/중앙) ← 거의 100% 1순위
2. 발병 시기 (오늘/어제/N일전/N주전/만성)
3. 통증 양상 (radiating, 야간통, 동작별, VAS)
4. 기전 / 악화 / 완화 요인
5. 과거 치료·검사 이력 (외부 MRI, 타원 진료, CD)
6. 동반 질환 / 직업·생활 (선택적, 고령일수록 더 많이 묻힘)
```

### 2. O 단계 — 이 의사 패턴

- **신체검진을 본문에 거의 안 적음** (LOM(+), Pus, Decomp state 정도만). 처방 코드(x-ray N매)로 영상 검사 여부가 드러남.
- 이는 **검진 결과를 음성/정상이면 생략**하는 차팅 스타일 → UI에서 "정상이면 자동 생략" 옵션이 효율적.

### 3. A 단계 — 진단 유형

| 유형 | 빈도 | 예시 |
|---|---|---|
| 확정 진단 | 5/10 | Frozen shoulder, Ankle sprain, Contusion 등 |
| R/O (감별) | 1/10 | Spinal stenosis -- r/o |
| 다중 진단 | 2/10 | Acute lumbar sprain + MPS, Frozen + 손목 건초염 |
| 수술 후 상태 | 1/10 | Ligament repair ankle Rt 후 |
| 골다공증/자세 | 1/10 | Decomp state / Flatback |

### 4. P 단계 — 거의 자동화된 템플릿

```
[표준 P 묶음 — 외래 정형외과 빈출]
├─ PT (간섭파 + 심층열 + 표층열) ← 7/10에서 등장, 사실상 디폴트
├─ PO 1wk (NSAIDs + 위장보호제 ± 근이완제)
├─ 시술 (필요시): 주사(SNRB/관절/신경차단), 체외충격파, I&D
├─ 보조기 / 주의사항
├─ F/U: 1wk 표준, 추적은 4-6wks
└─ 서류: 소견서, 진단서 (요청 시)
```

→ **UI에서 "표준 P 패키지"를 한 번에 적용 + 가감하는 형태가 의사의 실제 흐름과 일치.**

### 5. 반응형 트리거 후보 (10건에서 관찰된 시그널)

| 트리거 (S에서의 입력) | 의사가 다음에 묻는/적는 것 |
|---|---|
| 허리 + 기침 시 통증 | 다리 방사통/저림 → MRI 이력 (Case 1) |
| 어깨 + 야간통 | 동작별 통증 → LOM → Frozen shoulder (Case 2) |
| 어제/오늘 외상 | 기전 + 측 → x-ray로 골절 배제 (Case 4, 10) |
| 운동 직후 통증 | 종목 + 빈도 → x-ray ± 초음파 (Case 2, 10) |
| 외상 + 다리 방사 + 고령 | 동반질환 + 외부 영상 → R/O Spinal stenosis (Case 7) |
| 다부위 호소 | 부위별 SOAP 반복 (부위 탭 추가) (Case 6) |
| 내원 목적 = 처치 | 이전 회차 + 다음 회차 (Case 5) |
| 수술 후 외래 | 수술 병원/시점/수술명 + 현재 기능 제한 (Case 9) |
| 추적 방문 | 치료 경과 시점 + 호전 정도 (Case 8) |

### 6. CCF 스키마에 추가 제안되는 항목

10건을 분해하면서 현 스키마(`CCF_SOAP_charting_requirements.md`)에 **누락**된 것들:

- **내원 모드 분기**: 호소 / 추적 / 처치 / 수술 후 (현재는 호소만 가정)
- **타원 수술 이력 블록**: 수술 병원 / 시점 / 수술명
- **외부 영상/CD 등록**: 외부 MRI·CT 이력 (Case 1, 6, 7)
- **동반 질환 / 복용 약**: 고령·만성 환자 필수 (Case 7)
- **사회력/직업 필드**: 사무직, 운동 빈도, 음주·흡연 (Case 2)
- **다부위 탭 자동 추가**: 한 방문 내 여러 부위 (Case 6, 7)
- **"이번 방문은 정보수집"** 플래그: R/O만 두고 다음 방문에 확정 (Case 7)
- **"표준 P 패키지" 단축**: 간섭파+심층열+표층열+PO 1wk를 한 번에 (전 케이스)
