아래는 **Focal Pain(국소 통증) “부위 트리 + 측성(중앙/양측/좌/우) 허용값”**을 **부위명은 한글 + (의학 용어)**로, 그리고 **C2~C7 같은 범위 표기 없이 전부 개별 항목으로** 다시 정리한 버전이야.
(중요: 여기에는 **압통/유발통 같은 ‘소견’은 포함하지 않고**, **오직 “위치(부위)”만** 포함했어.)

## 체크(너가 바로 구현할 때 흔히 하는 실수 2개)

* **정중선 구조(극돌기, 치골결합, 미골 등)**에 `좌/우/양측`을 허용하면 데이터가 깨져. → 위처럼 **중앙만 허용**이 기본.
* **손/발의 관절**은 laterality(좌/우/양측) 외에 digit(손가락/발가락 번호)까지 같이 관리하는 게 정석인데, 지금은 요청대로 “부위 전부 노출” 형태로 풀어 썼어.

원하면 다음 단계로, 이 리스트를 그대로 기반으로 **(1) 노드 ID 체계**, **(2) 표시용 라벨 vs 저장용 코드**, **(3) laterality 기본값/숨김 규칙(예: 중앙 고정이면 UI에서 선택 숨김)**까지 한 번에 설계해줄게.
