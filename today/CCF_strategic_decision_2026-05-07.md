# CCF 전략 의사결정 — 과거 차트 마이닝 vs 클릭 차트 우선

> 작성일: 2026-05-07
> 맥락: `docs/CCF_mining_plan.md` 작성 후, "마이닝 ROI 정당성"에 대한 재검토
> 결론 요약: **클릭 차트 우선 출시 → 누적 데이터 기반 분기점 판단**

---

## 0. 출발점이 된 질문

> 병원 과거 차트를 기반으로 SOAP 데이터 구조화 → 클릭형 차트 제공.
> 클릭 차트의 시작 목적은 (1) 의사 사용성 (2) **데이터 구조화 저장 문제 해결**.
> 과거 차트 기반 SOAP 구조화가 ROI에 맞는가? 구조화 후 다른 아이템을 해야 하는가?

---

## 1. 질문 재구성 (가장 중요한 인사이트)

원래 질문은 두 가지를 묶고 있다:
- A) 클릭 차트의 사용성
- B) 데이터 구조화 저장

이 둘이 같은 솔루션에 묶여 있다는 가정이 깔려있는데, **실은 분리 가능**하다.

### 핵심 인사이트
> **"구조화된 데이터 저장" 문제는 클릭 차트 자체가 해결하는 문제다.**
> 과거 차트 마이닝은 그 위의 부트스트랩 가속이지, 본질적 해결책이 아니다.

이유:
- 클릭 차트는 출시 첫날부터 **구조화된 데이터를 생산**한다 (의사가 칩을 클릭하는 순간 = 이미 토큰화된 입력)
- 따라서 **베이스 템플릿(CCF 스키마)만으로도 클릭 차트는 출시 가능**
- 과거 차트 마이닝은 "초기 칩 후보 풍부화"의 부트스트랩일 뿐, 필수가 아님

이 인사이트가 의사결정의 출발점이다.

## 2. 세 시나리오의 ROI 비교

### 시나리오 A — 베이스 템플릿으로 클릭 차트 먼저, 마이닝 없음

| 항목 | 내용 |
|---|---|
| 투자 | 클릭 차트 UI + CCF 스키마 베이스 룰북 (6~8주) |
| 산출 | Day 1부터 구조화된 데이터 자연 누적, 6개월 후 자기 클릭 로그로 룰북 강화 가능 |
| 강점 | 가장 빠른 출시, 마이닝 비용 0, 구조화 문제 사실상 해결 (전향적) |
| 약점 | 콜드 스타트 의사 차별화 약함, "당신만의 차트" 영업 메시지 못 함 |

### 시나리오 B — 마이닝 → 클릭 차트

| 항목 | 내용 |
|---|---|
| 투자 | 마이닝 파이프라인(6주) + 클릭 차트(6주) + PHI 컴플라이언스 |
| 산출 | 의사별 개인화 첫날부터, 차별화 메시지 강함 |
| 강점 | 도입 동기 ↑, 락인 자연 형성 |
| 약점 | 18~24개월 투자, 단위 경제 박함 (의사당 변동비 30만원), PHI 부담 무거움 |

> **보이는 것보다 큰 함정**: 클릭 차트만으로도 시간이 흐르면 같은 효과. 자기 클릭 누적 = 자기 패턴 자동 학습. 마이닝의 가치는 **그 시간을 앞당기는 것뿐**이다.

### 시나리오 C — 과거 차트 구조화를 "수평 역량"으로 (멀티프로덕트)

| 항목 | 내용 |
|---|---|
| 투자 | 마이닝 + 클릭 차트 + 보험청구 자동화 + 검색/분석 + ... |
| 산출 | 구조화 데이터 위에 여러 제품 라인 |
| 강점 | 마이닝 비용 amortize, 각 제품의 ROI 누적 |
| 약점 | 멀티프로덕트는 팀 규모 필요, 초기 도입 진입점 분산 |

## 3. 솔직한 평가

### 클릭 차트만 놓고 보면 → 시나리오 A가 ROI 압도

마이닝의 "개인화" 가치는 실제로 **클릭 차트 사용 6개월이면 자기 클릭 로그로 자연 달성**된다.
- 그 6개월 vs 마이닝 18~24개월 투자 → 마이닝이 더 비쌀 가능성
- 베이스 템플릿이 충분히 잘 만들어지면 콜드 스타트도 견딜 만함
- 클릭 차트의 본래 가치(사용성 + 구조화 저장)는 시나리오 A로도 달성

### 마이닝이 정당화되는 경우 → 시나리오 C 관점일 때만

과거 차트 구조화의 진짜 가치는 클릭 차트가 아니라:

| 후보 제품 | 가치 명제 | ROI 명확성 |
|---|---|---|
| **보험 청구 자동화** | 진단·처방 구조화 → 청구 코드 자동 매핑 → 의사·청구사 시간 절감 | **매우 높음** (B2B 의료 SaaS의 고전적 해자) |
| 차트 검색·필터 | "지난달 동일 진단 환자 처방" 즉시 조회 | 높음 |
| 품질·통계 대시보드 | 클리닉 운영진 대상 (다른 페르소나) | 중간 |
| 연구·분석 데이터셋 | 비식별 코호트 → 별도 사업 | 중간 (긴 사이클) |

이 중 **하나 이상을 사업 라인으로 가져갈 계획이 있으면** 마이닝 투자가 정당화된다.
**클릭 차트만 본다면 마이닝은 사치에 가깝다.**

## 4. 권장 순서

```
[1단계] 베이스 템플릿 클릭 차트 (3개월)
  ├─ 목적: 빠른 출시, 사용성 검증, 의사 도입
  └─ 데이터: Day 1부터 클릭 = 구조화 데이터 자연 누적

[2단계] 3~6개월차 분기점 판단 (누적 데이터 기반)
  ├─ 콜드 스타트가 실제 의사 이탈 원인인가?
  ├─ 의사들이 추가로 원하는 기능은 무엇인가?
  └─ 클리닉 운영진의 페인 포인트는?

[3-A] 콜드 스타트가 핵심 문제 → 과거 차트 마이닝 도입 (시나리오 B)
[3-B] 청구·검색·분석 요청 다수 → 마이닝을 수평 역량으로 확장 (시나리오 C)
[3-C] 클릭 차트 자체가 충분 → 마이닝 보류, 다른 부위/과목으로 확장
```

핵심: **의사결정을 데이터가 쌓일 때까지 미룬다.** 지금 베팅하지 않는다.

## 5. 한 가지 Caveat — 의사 인터뷰 필수

시나리오 A의 가정은 "**의사가 베이스 템플릿 + 누적형으로 만족한다**"는 것이다.
이 가정이 깨지는 경우:

> 영업 메시지에서 **"당신만의 차트"가 도입의 핵심 동인**이라면, 시나리오 A는 약하다.
> 일반 EMR 템플릿과 차별화 없는 첫인상으로는 도입이 안 될 수 있다.

### 검증 방법
**의사 5~10명 인터뷰**:
- "베이스 템플릿 + 시간이 지나면 당신 패턴으로 진화" vs "첫날부터 당신 차트 패턴 그대로"
- 어느 쪽에 더 큰 가치를 두는가?
- 도입을 결정하는 결정적 요인은 무엇인가?

이 답이 시나리오 A vs B 사이의 **전략 분기점**이다. 인터뷰 없이 결정하면 가설 베팅이 된다.

## 6. 최종 권장

### 단기 (지금~3개월)
1. **시나리오 A로 출시**: 베이스 템플릿 + 클릭 차트 빠르게
2. **의사 인터뷰 5~10명**: 시나리오 A vs B 가치 검증
3. **클릭 로그 인프라 동시 구축**: 출시일부터 데이터 누적 시작
4. **PHI 처리 방침 사전 결정**: 향후 마이닝 도입 시 발목 잡지 않도록 컴플라이언스 인프라는 선제 검토

### 중기 (3~6개월)
5. **누적 사용 데이터 + 의사 피드백으로 분기 결정**
6. **선택지 보류**: 마이닝 도입 / 멀티프로덕트 확장 / 다른 부위 확장 중 선택

### 장기 (6~12개월)
7. 위 결정에 따라 자원 집중

## 7. 이미 작성한 마이닝 문서의 위치

**`docs/CCF_mining_plan.md`는 폐기하지 않는다.**

이유:

- 시나리오 B/C로 갈 경우 그대로 실행 가능
- 마이닝의 한계와 사업성 분석은 의사결정 자체에 도움이 됨

위치 변경 제안:
- 현재: 실행 계획서 위치
- 향후: **"옵션 검토 자료"** 위치로 격하 — 결정이 필요한 시점에 다시 꺼낸다
